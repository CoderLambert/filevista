# 16 阶段完整实现

## 1. 新增 CI workflow

新增文件：

```txt
.github/workflows/ci.yml
```

完整内容：

```yaml
name: CI

on:
  pull_request:
    branches:
      - main
      - dev
  push:
    branches:
      - main
      - dev

permissions:
  contents: read

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  validate:
    name: Lint, Test, and Build
    runs-on: ubuntu-latest

    timeout-minutes: 15

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Print tool versions
        run: |
          bun --version
          node --version

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Run lint
        run: bun run lint

      - name: Run tests
        run: bun run test:run

      - name: Build application
        run: bun run build
```

这个版本足够稳，暂时不要加缓存，也不要做矩阵多版本。原因是当前目标是“工程基线”，不是复杂 CI 优化。

## 4. 建议新增类型检查脚本，但先不要接入 CI 阻塞

我注意到你的 `next.config.ts` 里设置了：

```ts
typescript: {
  ignoreBuildErrors: true,
}
```

也就是说，`next build` 当前不会因为 TypeScript 类型错误失败。

16 阶段可以先加一个非 CI 必跑的脚本：

```json
"typecheck": "tsc --noEmit"
```

然后本地需要时手动跑：

```bash
bun run typecheck
```

但是我不建议马上把它放进 `check` 或 CI，因为你当前 Next 配置显式忽略 build type errors，说明项目里可能已有历史类型债。直接强接 CI 可能会让阶段目标扩大。

最终 scripts 推荐：

```json
{
  "scripts": {
    "dev": "next dev -p 3000 2>&1 | tee dev.log",
    "build": "next build && cp -r .next/static .next/standalone/.next/ && cp -r public .next/standalone/",
    "start": "NODE_ENV=production bun .next/standalone/server.js 2>&1 | tee server.log",
    "lint": "eslint .",
    "typecheck": "tsc --noEmit",
    "copy:pdf-worker": "node scripts/copy-pdf-worker.mjs",
    "postinstall": "node scripts/copy-pdf-worker.mjs",
    "db:push": "prisma db push",
    "db:generate": "prisma generate",
    "db:migrate": "prisma migrate dev",
    "db:reset": "prisma migrate reset",
    "test": "vitest",
    "test:run": "vitest run",
    "test:watch": "vitest --watch",
    "check": "bun run lint && bun run test:run && bun run build"
  }
}
```
